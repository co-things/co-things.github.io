linux后台执行任务技巧

发表于 2015-09-09   |  

有时候我们需要远程登录到linux执行一些长时间的任务，通常情况下我们都是为每一个这样的任务开一个远程终端窗口，由于他们执行的时间太长了。必须等待它执行完毕，在此期间可不能关掉窗口或者断开连接，否则这个任务就会被kill掉，一切半途而废了。

解决办法

当用户注销或网络断开时，终端进程会收到HUP信号，导致关闭其所有子进程。所以解决办法有2种，一种是让进程忽略HUP信号，第二种是让进程不属于终端进程的子进程

使用nohup命令

nohup的用途就是让进程忽略HUP信号，属于第一种解决办法

NOHUP(1)                                                 User Commands                                                NOHUP(1)

NAME
       nohup - run a command immune to hangups, with output to a non-tty

# nohup ping www.baidu.com > /dev/null 2>&1 &
[1] 13210
# ps -ef | grep ping
root     13210 12621  0 21:01 pts/22   00:00:00 ping www.baidu.com

默认情况下，标准输出和错误输出会重定向到nohup.out文件上

使用setsid命令

在一个新的会话中运行命令,属于第二种解决办法

SETSID(1)                                                User Commands                                               SETSID(1)

NAME
       setsid - run a program in a new session

# setsid ping www.baidu.com > /dev/null 2>&1
# ps -ef | grep ping
root     13260     1  0 21:03 ?        00:00:00 ping www.baidu.com

此时进程的父进程id变成init进程id，而不是终端进程的id

使用screen命令

SCREEN(1)                                           General Commands Manual                                          SCREEN(1)

NAME
       screen - screen manager with VT100/ANSI terminal emulation

常用命令
screen -dmS 会话名 :建立一个处于断开模式下的会话
screen -list :列出所有会话
screen -r 会话名 :重新连接指定会话
快捷键CTRL-a d :暂时断开当前会话

# screen -dmS mysession
# screen -list
There is a screen on:
        15966.mysession (09/09/2015 09:43:06 PM)        (Detached)
        1 Socket in /var/run/screen/S-root.

# screen -r mysession
# ping www.baidu.com > /dev/null 2>&1
# pstree
init─
     ├─screen───zsh───ping
     ├─snmpd
     ├─sshd─┬─sshd───zsh
     │      └─sshd───zsh───tmux

可以看到ping进程并不是sshd的子进程，所以该方法属于第一种解决办法
